أريد إعادة تنظيم المشروع وتحسينه بشكل احترافي بناءً على المعمارية الحالية الموجودة في الكود.
بعد مراجعة المشروع، واضح أن النظام الحالي قوي كبداية، لكن يحتاج إعادة ترتيب حتى يصبح أنظف، أسرع، وأسهل للتوسع.

التحديث 1: تحويل الواجهة من React + Vite إلى Next.js

المشروع الحالي مبني على React + Vite + Wouter، لكن أريد تحويل الواجهة إلى Next.js App Router.

الهدف من هذا التحويل:

تنظيم أفضل للصفحات والـ layouts
بنية أوضح للويب
تحسين rendering
تحسين تنظيم الـ routes
تجهيز المشروع ليكون أقرب لمنتج حقيقي
تسهيل SSR / streaming UI مستقبلًا
تحسين فصل الواجهة عن منطق الوكيل

المهم:
Next.js يكون مسؤول عن:

واجهة الشات
صفحات الإعدادات
صفحات MCP
الـ layout
طبقة الويب
route handlers / BFF عند الحاجة

ملاحظات مهمة من مراجعة الكود
المشروع الحالي ليس Next.js، بل React + Vite.
المشروع الحالي يعتمد بشكل كبير على TypeScript أكثر بكثير من Python.
Python الحالي ليس بعد Agent Core كامل، بل gateway محدود.
المشروع فيه ملفات كبيرة جدًا تحتاج تفكيك.
الـ naming الحالي ما زال مربوطًا بـ Anthropic أكثر من اللازم.
الـ Terminal قوي لكنه يحتاج أمان وتنظيم أعلى.
الـ attachments تحتاج تصميم جديد.
إعدادات المشروع تحتاج structured config أفضل.
الـ health checks تحتاج أن تكون حقيقية وليست سطحية.
المشروع عنده أساس ممتاز، لكن الآن يحتاج مرحلة Product Architecture حقيقية.
