تحديث جديد مطلوب على صفحة MCP Servers

أريد إضافة مساعد جانبي ذكي داخل صفحة MCP Servers نفسها، وليس شات عام منفصل.
الفكرة: في الجهة اليمنى أعلى الصفحة يكون هناك زر/أيقونة، وعند الضغط عليه يظهر Panel / Sidebar من اليمين مع Animation ناعم.

المطلوب من هذه الخانة الجانبية:

هذه الخانة تكون عبارة عن محادثة مباشرة مع الوكيل/الذكاء الاصطناعي لكن مخصصة فقط لهذه الصفحة، يعني الوكيل يفهم أن المستخدم الآن داخل صفحة إدارة خوادم MCP، ويستطيع يساعده بناءً على العناصر الموجودة داخل الصفحة.

قدرات الوكيل داخل هذه الخانة:

الوكيل لازم يقدر:

يقرأ الخوادم الموجودة حالياً في الصفحة.
يعرض سجل/ملخص عن الخوادم أو العمليات التي تمت.
يعدل على إعدادات أي Server موجود.
يضغط الأزرار الموجودة داخل الصفحة بالنيابة عن المستخدم بعد تأكيد واضح.
ينشئ Server جديد.
يملأ الحقول تلقائياً داخل نموذج الإضافة أو التعديل.
يقترح إعدادات مناسبة حسب نوع السيرفر.
يفهم أوامر اللغة الطبيعية مثل:
“أضف سيرفر جديد”
“عدّل Test Server”
“غيّر Authentication إلى Bearer Token”
“خلي هذا السيرفر معطل”
“أنشئ سيرفر جديد بنفس إعدادات هذا لكن باسم مختلف”
سلوك مهم جداً:

الوكيل لا يكون فقط شات نصي، بل يكون مرتبط فعلياً بعناصر الواجهة.
يعني لو المستخدم طلب تعديل أو إنشاء أو ضغط زر، الوكيل يقدر ينفذ ذلك داخل الصفحة نفسها.

نقطة مهمة في التجربة:

إذا الوكيل يريد ينفذ Action مهم مثل:

حذف Server
تعديل إعدادات حساسة
تعطيل/تفعيل Server
حفظ تغييرات

لازم يظهر تأكيد واضح للمستخدم قبل التنفيذ.
مثلاً:

Confirm
Cancel
أو اقتراحات جاهزة داخل الشات
واجهة الشات نفسها:

أريد واجهة الشات تكون حديثة وواضحة، وفيها الحالات التالية:

1) أثناء الكتابة

إذا الوكيل يفكر أو يكتب، يظهر:

typing indicator
أو animation توضح أن الوكيل يعمل الآن
2) أثناء تنفيذ مهمة

إذا الوكيل ينفذ شيء داخل الصفحة، يظهر داخل الشات:

حالة التنفيذ الحالية
مثل:
“جارِ قراءة الخوادم”
“جارِ فتح نموذج التعديل”
“جارِ تحديث الإعدادات”
“تم حفظ التغييرات”
3) أثناء البحث أو التحليل

إذا الوكيل يحلل الحالة أو يبحث بين الخوادم، يظهر له صندوق حالة/Status Box داخل المحادثة، يوضح للمستخدم ما الذي يحدث الآن، بدل ما تكون المحادثة صامتة.

التصميم المطلوب:
الـ panel يفتح من اليمين بـ animation سلس.
لا يغطي الصفحة بشكل مزعج، لكن يكون واضح ومريح.
يكون شكله متناسق مع الثيم الحالي الداكن.
المحادثة تكون مخصصة لهذه الصفحة فقط.
داخل الشات ممكن تظهر Cards / Action Blocks / Suggested Actions بدل النص فقط.
لو الوكيل يقترح شيء، يظهر للمستخدم أزرار جاهزة مثل:
تعديل الآن
إنشاء Server
حفظ
إلغاء
عرض السجل
التكامل المطلوب:

أريد هذا المساعد يكون مرتبط مباشرة بمنطق الصفحة:

قراءة حالة السيرفرات الحالية
معرفة هل السيرفر connected أو لا
الوصول إلى النماذج الموجودة
تعبئة الحقول
تنفيذ الأزرار
تحديث الواجهة مباشرة بعد كل عملية
أمثلة استخدام:

المستخدم ممكن يقول للوكيل:

“أضف MCP Server جديد باستخدام HTTP Remote”
“عدّل Test Server وخلي الـ timeout 60”
“أنشئ نسخة من Smoke Test MCP Server”
“اعرض لي كل الخوادم المتصلة”
“احذف السيرفر هذا”
“افتح إعدادات هذا السيرفر”
“وش أفضل Authentication لهذا endpoint؟”
المطلوب تقنياً:

أريد التنفيذ يكون مبني بطريقة منظمة:

Agent sidebar / right drawer
UI state integration
action execution layer
confirmation flow
chat state + progress state + action history
natural language to UI actions
logs/history داخل نفس الخانة
الهدف من التحديث:

أريد صفحة MCP Servers تتحول من صفحة إعدادات عادية إلى واجهة تفاعلية ذكية، بحيث المستخدم يقدر يدير كل شيء من خلال الكلام الطبيعي، والوكيل ينفذ داخل الصفحة نفسها بشكل مباشر وواضح وآمن.
