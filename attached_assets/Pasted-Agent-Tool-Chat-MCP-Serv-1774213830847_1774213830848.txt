Agent Tool Chat
الاسم قابل للتغيير، لكن الفكرة الأساسية هي:
منصة شات احترافية لإدارة وتشغيل MCP Servers، مع وكيل ذكي ينفذ المهام عبر الأدوات المتصلة، ويعرض النتائج داخل واجهة محادثة مرتبة وواضحة.
1) فكرة المشروع الأساسية
المشروع عبارة عن تطبيق ويب فيه:
واجهة شات رئيسية كبيرة
وكيل ذكي داخل الشات
إمكانية ربط أي MCP Server
اكتشاف الأدوات والموارد المدعومة تلقائيًا
تنفيذ المهام عبر هذه الأدوات
عرض النتائج داخل الشات بشكل منظم
لوحة إعدادات لإدارة السيرفرات، الأدوات، الصلاحيات، وحالة الاتصال
الهدف من المشروع ليس مجرد “شات”، بل مركز تشغيل Agent + MCP.
يعني المستخدم بدل ما يدير الأدوات يدويًا، يربط السيرفرات مرة واحدة، وبعدها الوكيل يشتغل عليها بشكل مرتب وتحت سيطرة واضحة.
2) الهدف من المشروع
الهدف هو بناء نظام يسمح للمستخدم أن:
يربط أي MCP Server محلي أو خارجي
يرى حالة الاتصال لكل سيرفر
يرى الأدوات التي يدعمها كل سيرفر
يرسل مهمة بلغة طبيعية داخل الشات
الوكيل يقرر أو المستخدم يقرر أي أداة تُستخدم
يتم التنفيذ
تظهر النتيجة بشكل واضح داخل المحادثة
تكون هناك سيطرة كاملة على التنفيذ والصلاحيات
3) فلسفة المشروع
المشروع مبني على مبدأ مهم:
الذكاء الاصطناعي لا يسيطر وحده
بل يوجد طبقة تحكم واضحة فوق الوكيل.
هذا يعني:
الوكيل لا ينفذ أي شيء عشوائيًا
توجد طبقة سياسات وصلاحيات
توجد حالة تشغيل واضحة
توجد موافقة على الأدوات الحساسة
توجد حدود زمنية ومحاولات retry
توجد سجلات logs لكل شيء
هذه الفكرة مهمة جدًا لأنك تريد نظام قوي واحترافي وليس شات فوضوي.
4) التقنيات المعتمدة للمشروع
الواجهة الأمامية Frontend
Next.js
TypeScript
Tailwind CSS
shadcn/ui
Vercel AI SDK لواجهة الشات والـ streaming
Next.js App Router هو الخيار الصحيح لواجهة الشات الحديثة، ويدعم Server/Client Components وstreaming بشكل رسمي.
الباكند Backend
Python 3.11+ أو 3.12
FastAPI
Pydantic
Uvicorn
FastAPI مناسب جدًا هنا لأنه سريع، مبني على type hints، ويدعم WebSockets وbackground tasks وstreaming responses رسميًا.
طبقة الوكيل Agent Layer
PydanticAI
PydanticAI مناسب لأنك تريد طبقة Agent في بايثون مع مرونة وتحكم عالي، وهو يدعم MCP clients ويعطيك طريقة نظيفة لبناء system prompts، tools، والتحكم في الرسائل. كما أن سياسة الإصدارات توضح أن V1 مستقر ولن يحصل فيه كسر API في الإصدارات الثانوية.
طبقة MCP
MCP Python SDK الرسمي
ويمكن إضافة FastMCP لاحقًا إذا احتجنا طبقة أسرع لبناء أدواتنا
مواصفة MCP نفسها تبني النظام حول أدوات Tools وموارد Resources وPrompts، مع transports وبروتوكول واضح للتكامل.
قاعدة البيانات

الوظائف الخلفية
Celery أو بديل أخف لاحقًا

Logs منظمة داخل النظام
الأمان
تشفير الأسرار
OAuth 2.1 للسيرفرات الحساسة إذا لزم
Allowlist للدومينات

MCP عنده إرشادات واضحة للأمان والتفويض، بما فيها OAuth 2.1 لحماية الموارد والعمليات الحساسة.
